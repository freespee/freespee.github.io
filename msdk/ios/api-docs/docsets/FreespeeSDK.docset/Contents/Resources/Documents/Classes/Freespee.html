  • -connectIdentifier:completion:

    This function will connect to the SDK backend using the given identifier. It will also register for VoIP-push and request audio recording permissions from the user unless already granted.

    Emitted errors: FreespeeErrorInvalidState FreespeeErrorInvalidAPIKey FreespeeErrorInvalidArgument FreespeeErrorAudioRecordingPermission FreespeeErrorNetwork

    Parameters

    identifier

    The identifier to connect.
    completion

    Called upon completion with an error if something went wrong.

  • -startListeningForIncomingCalls:

    Sets the SDK in a state where it listens for incoming phone calls without the need for recieving a VoIP push notification.

    Note

    You should use this for debugging/testing purposes only, using this in production will consume significant battery life.

    Emitted errors: FreespeeErrorInvalidState FreespeeErrorInternal

    Resolving block to get a localized name to display for a Freespee call in CallKit. Without a resolver set all calls will be showing off as Unknown. Typically you will use the peerIdentifier on the FreespeeCall to do your own lookup as it is an identifier known to the app domain.
